Through the Ages
Game Design Document

by Jakub Wlodek, Nick Pirello, Vinny Paladino

INTRODUCTION

This document describes a game called "Through the Ages", which is designed to be a 2D platformer and action game. It will use sprites for animation, a grid system using 'Tiled' for backgrounds, and phaser for physics, collisions, and other game programming functions.


TECHNOLOGY

Through The Ages will be developed for web browsers using the Phaser game engine which is a Desktop and Mobile HTML5 game framework created by Richard Davey (Photonstorm). WebGL primarily handles the rendering for the Phaser framework. Audio and music will be created in Audacity. Tiled will be used to create the levels. Sprites and tilesets will be relatively simplistic and will created in Fire Alpaca.


BACK STORY

Through the Ages will attempt to recreate a span of history. The player, who controls Gareth I and his descendants through the ages, moves through each levels by collecting objects that will help move human civilization into the modern age. Once he has all of the necessary technology and supplies, he moves forward through history, all while his enemies try to defeat him and halt human progress. Can Gareth overcome all odds and move humanity into the future?


OBJECTIVE

Gareth's task is to explore the world and find and collect all of the technological advancements of his current age. Once he has collected one he must bring it back to his family. Once Gareth's family has possesion of all the items the can safely survive until the next age. Unfortunatly for Gareth these items are scattered through out the world and he only has the space in his pockets for one item at a time.

The world that Gareth and his family live in is a very dangerous place. The local wildlife have taken a liking to the items and will go to any means to keep them. This means that Gareth will have to protect himself using the weapon of the current age that has been given to him by his family.


GAMEPLAY

The game will be a 2D platformer with several levels. Gareth will be able to run side to side, leap up and down, double jump, etc. Each level will have a collection of items scattered around it that are required to reach the next milestone in human progress (ex. Fire in pre historic times, gunpowder/telescope in medieval times etc.).

This process will not be unopposed, however, as Gareth will be opposed at each step of the way by the enemies of humanity during each time period, from dinosaurs to aliens. In each era, Gareth will find that the items are placed in a variety of different locales, and the locations of the items and the items themselves will be different as well. Gareth will fight these enemies and attempt to collect all of these together, pushing humanity further along. Luckily, Gareth will have some help along the way:

  • Evolving technology means evolving armor and weapons as well. Gareth can better defend himself in each consecutive level.
  • Power ups will randomly appear and will be unique to each time period. For example, in prehistoric times, when Gareth is attacked by dinosaurs, a meteor strike can be used to defeat them all.
  • Once his civilization has been built up substantially, Gareth can employ the aid of other citizens or technologies.

Gareth will collect items in the wild and return them to his ever growing city, gradullay improving its technology. Eventually, the city is able to progress further through history, and Gareth I passes the torch to Gareth II who passes it to Gareth III, and so on.


CONTROLS

The game will be controlled exclusively with the keyboard, with a similar control scheme to most platformer games

  • A, D, Arrwy Keys - Will move Gareth left and right along the current platform/floor
  • Spacebar - Makes Gareth jump
  • Right Shift - When by an item and no item in his pocket, Gareth will pick up the item. If by his home, and with an item in his pocket, Gareth will drop the item off.
  • Right Enter - Gareth attacks in the direction he is facing.
  • Right Control - Provided he has a special attack token in his pocket, Gareth will execute his special attack.
  • Escape - Will pause the Game

GRAPHICAL USER INTERFACE

As far as the GUI is concerned, if we consider

  • Splash Screen - The splash screen GUI simply presents a game logo and a Start button for the user to press when they are ready to play. Upon pressing it, a brief scripted sequence is played explaining a little about Gareth, his family and the back story for the game. While presenting the message "Press Enter to Skip" should be displayed at the bottom of the screen. If pressed, the game should immediately start.

  • In-Game Menu - While the game is in progress, we'll always have a game menu that allows the player to select from the following options:
    • New Game - If a game is in progress, first we'll pop up a dialog to make sure the user really wants to quit the current game. If we conclude the user really wants to start a game, then the game state is reset and begun.
    • Pause Game - This button is deactivated when a game is not in progress. If pressed, it will produce the same effect as the ESC key.
    • Display Controls - If pressed, the game will display an info screen that includes a description of all game controls.
    • Best Times - If pressed, the game will display an info screen with the fastest times for clearing the level.
    • About - If pressed, the game will display an info screen with information about the game author.
    • Exit - If pressed, the game program will exit.

  • In-Game GUI - Once a game starts, Gareth will walk into view from the left side at the bottom of the level. Once he reaches his starting point the level's monsters will move around and try to attack Gareth if he gets too close to the items. While playing the following will be displayed at all times:
    • Time Elapsed
    • Items delivered
    • Item currently held
    • A health bar

ARTWORK

We will create the following artwork for Through the Ages

  • Gareth and his descendants Gareth and his descendents share a striking resemblance, in both name and looks. Gareth I looks the same as Gareth XXIV, aside from the time appropriate armor each of them dons.
    • Walking animations
    • Jumping animations
    • Collect and Drop item
    • Use power up
    • Attacking Animations
    • Death Animations

  • Enemies - Dinosaurs, enemy knights, evil robots etc. Gareth's challengers
    • Walking animations
    • Attacking animations
    • Death Animations

  • Gareth's City - Gareth's city will expand in size and will progress through history during each level.

  • Platforms - Each level will contain a variety of platforms and barriers around which Gareth must jump to find his necessary items. These will range from glaciers during the ice age, to mountains, etc.

  • Backgrounds - Each era will have a background tileset that reflects the era in history: ex. A gothic castle for the middle ages.
